# Handoff Sprint 5 → Sprint 6

## Objetivo

Este paquete permite iniciar el Sprint 6 de Deployment sin bloqueos técnicos.

## Artefactos principales

- `models/final_model.pkl`: modelo final validado en Sprint 4.
- `notebooks/15_business_value.ipynb`: evaluación de valor de negocio.
- `reports/business_value_summary.csv`: resumen económico.
- `reports/threshold_business_value.csv`: evaluación de umbrales.
- `reports/gain_curve.csv`: datos para curva de ganancia.
- `reports/lift_curve.csv`: datos para curva lift.
- `reports/business_value_sensitivity.csv`: escenarios de sensibilidad.

## Uso esperado en Sprint 6

El equipo de Deployment debe empaquetar el modelo como API o integrarlo a un dashboard desplegable, usando el umbral óptimo definido en el análisis de negocio.

## Decisión

El modelo puede pasar al Sprint 6 como MVP controlado, sujeto a validación final del sponsor.
